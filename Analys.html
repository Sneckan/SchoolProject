  • Tidshantering

    Eftersom jag vet att jag har svårt att "författa" texter men att jag har lätt och är snabb på att göra tekniska grejer (så som att skriva kod, komma med lösningar till tekniska problem, etc), så planerade jag upp så att jag hade så mycket tid som möjligt med att jobba med texterna på sidorna. Dessutom tänkte jag att det är affärsmessigt sett viktigt att se till att sidfunktionerna är klara först.

  • Affärsvärdes övervägande

    Som jag skrev i tidshanteringen så prioriterade jag att fixa funktionaliteten hoss sidorna först, delvis att jag kände att jag behövde all möjlig tid för att jobba på sidinnehållet, men också för att göra det lättare om innehålls-ändringar behövs. Dessutom gör det även att (om detta var ett ricktigt uppdrag från ett företag) om projektet skulle ges till någon annan, kan hen lätt plocka upp projektet och fylla på innehållet.

  • Tekniska beslut

    Jag känner att på grund av hur kravspecen var uppsatta hade jag inget behov av att göra några tekniska beslut, utöver möjliga designval med tekniska lösningar. Sen om det fanns möjligheter att göra det så var jag i sånna fall inte medveten om dem.

  • Övrigt

    Jag kunde satt upp mina trello kort bättre och delat up dem i mindre kort för varje arbetsuppgift, så jag kunde ha bättre koll över vad jag hade kvar att göra. Så som jag hade dem nu ledde till att det blev lätt att säga "Jag har bara den här delen att göra, jag är nästan klar" när det egentligen var mer än bara "den delen" att göra.
