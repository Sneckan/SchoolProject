Klient-server-modellen

Klient-server-modellen är en model över hur olika programkomponenter kommuniserar med varandra. Klienten kännetecknas av att den frågar om information från servern, och servern ger infromation när tillfrågad av klienten. Ett exempel på klient-server-modellen är webbsidor, där klienten är webläsaren och servern är sidans servervärd.

Internetkommunikation


Klipp om hur internet fungerar
Internet är ett enormt nätverk av datorer/servrar som håller information. För att kunna kommunisera med andra servrar i nätverket måste de följa samma protokoll (se nedan). Detta protokoll kallas IP När information skickas mellan datorer över internet bryts de ner till mindre paket, som sedan byggs ihop igen när de kommit till rätt dator. För att paketen ska kunna hitta till rätt dator följer de en IP-Adress, en unik serie av nummer för varje dator på nätverket.
Vanligtvis är det för svårt för användare att komma ihåg IP-adresser till internetservrar, därför använder man sig av DNS servrar. En DNS server är en server som innehåller en databas med domännamn till IP-adresser, så att man lätt kan koppla upp sig till en server genom att bara komma ihåg dess namn.

Protokoll

Ett protokoll är en sorts kommunikations-överenskommelse mellan två eller flera deltagare, regler som alla flöjer för att undvika missförstånd.

HTTP/HTTPS

HTTP är ett kommunikationsprotokoll som används för att överföra webbsidor över internet. Det fungerar genom att klienten skickar en förfrågan till servern via en sträng som innehåller vilken HTTP version den använder och vilken fil klienten vill att servern ska skicka.
HTTPS fungerar på samma sätt men vid uppkopling skapas det en "sessionsnyckel" som används för att kryptera informationsöverföringen mellan klient och server.

FTP/FTPS

FTP är ett filöverföringsprotokoll som använder två separata uppkoplingar, en för kommandon och en för dataöverföring. Används oftast till att ladda upp webbsidor till servrar.
FTPS fungerar som FTP men den skapar en säker uppkopling genom två metoder.

  • Implicit uppkoppling

    Klienten "utmanar" servern med ett krypterat "hallå" meddelande. Om servern inte får ett matchande "hallå" bryts uppkoplingen direkt.

  • Explicit uppkoppling

    Klienten måste fråga om en säker uppkoppling till servern, som sen sätter upp en överenskommen kryptionsmetod. Om klienten inte gör en säkerhetsförfrågan kan servern låta klienten fortsätta i "osäkert" läge eller vägra uppkopplingen helt och hållet.

Tillgänglighet

WCAG är en lista av riktlinjer för att göra hemsidor tillgängliga för alla.

    Några exempel punkter från WCAGs hemsida:

  • 1.4.1 Use of Color

    Color is not used as the only visual means of conveying information, indicating an action, prompting a response, or distinguishing a visual element.

    Om någon som är färgblind går in på en sida där information ges ut endast via färg leder till att den färgblinde kanske inte kan ta upp den informationen.

  • 2.3.2 Three Flashes

    Web pages do not contain anything that flashes more than three times in any one second period.

    Om något blinkar mycket på sidan kan det trigga epilepsi.

  • 3.1.1 Language of Page

    The default human language of each Web page can be programmatically determined.

    Om man har "programmerat" vilket språk sidan följer kan de flesta webläsare översätta sidans text till användarens språk, vilket leder till att fler kan besöka och ta användning av sidan.
